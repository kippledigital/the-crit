The Crit — Product Requirements Document (PRD)

Product Name

The Crit

One-Liner

An AI-powered design feedback platform helping designers improve their craft through clear, educational, and actionable critique.

Problem Statement

  • Designers — especially students, juniors, and freelancers — struggle to get quality, structured feedback on visual and web design work.
  • Existing feedback options are:
    • vague and generic
    • scattered across platforms
    • slow or unavailable
  • Designers crave fast, trustworthy critique to:
    • improve design skills
    • build better portfolios
    • deliver professional-quality work
    • learn why decisions work or don’t

Solution

The Crit provides:

✅ AI-Generated Critique Across Disciplines

  • Visual design (graphic, brand, layout, typography)
  • UX/UI design
  • Web design analysis (including UI, layout, navigation, visual hierarchy)
  • Accessibility and usability insights
  • Copywriting feedback for interfaces and visuals

✅ Educational Focus

  • Critiques written in a helpful, mentoring voice
  • Explanations of why something should change
  • Encourages design growth, not just checklists
  • Example Crit Sheets users can read before submitting

✅ Multi-Agent Crit Engine

  • Orchestrated via n8n
  • Different “Crit Agents” handle:
    • visual feedback
    • UX/UI assessment
    • brand consistency
    • web layout and responsive design
    • accessibility
    • writing & microcopy critique
  • Agents collaborate into a single, unified Crit Sheet

✅ Delightful User Experience

  • Animated, accessible UI for file uploads
  • Upload visuals (images, PDFs, Figma .fig files, screenshots of web designs)
  • Input project goals, challenges, or rubrics
  • Clear error handling and friendly messaging
  • Option to view Sample Critique for trust-building

✅ Crit Sheet Output

  • Structured, scannable feedback
  • Praise for strengths
  • Actionable steps for improvement
  • Optional scoring against rubrics

User Personas

🎓 Student Designer

  • Learning visual or web design
  • Wants mentorship-style feedback
  • Preparing portfolio pieces for job hunting

💼 Freelance / Junior Designer

  • Works solo
  • Needs external perspective
  • Submits client work for critique

🧑‍🏫 Design Educator / Mentor

  • Reviews student work
  • Wants scalable, high-quality feedback
  • Interested in integrating Crit into coursework

Educational Mission

  • The Crit isn’t just a critique tool; it’s a learning partner.
  • Crit Sheets:
    • Explain design principles behind suggestions
    • Avoid harsh or purely negative feedback
    • Encourage iterative improvement
  • Vision: Help designers learn why decisions work, not just what to change

Core Features

  • File upload (images, PDFs, .fig)
  • Project metadata:
    • Title
    • Description
    • Design goals
    • Optional rubric
  • Animated file UI:
    • framer-motion for animations
    • accessible labels and error messages
  • Multi-agent critique engine
  • Vision model support (Claude-3 Vision, GPT-4 Vision, etc.)
  • Crit Sheet generation
  • Example critique call-to-action
  • Potential “Designers Online” social proof UI

Non-Functional Requirements

  • Fast response times
  • Secure file storage (e.g. Supabase buckets)
  • Privacy of user uploads
  • Cost-efficient AI usage (token management)
  • Accessibility-compliant UI

Tech Stack

  • Frontend: Next.js (or similar), React, Tailwind, shadcn/ui
  • Animations: Framer Motion
  • AI Models: Claude-3 Vision, GPT-4 Vision
  • Automation / Orchestration: n8n as the Crit Machine backend
  • Storage: Supabase for file storage
  • Docs / Knowledge Base: Notion for crit sheet templates, internal documentation

MVP Scope

✅ Upload designs and project description

✅ Process files and text via multi-agent AI

✅ Generate structured Crit Sheet

✅ Display Crit Sheet in UI

✅ View Example Crit Sheet

✅ Robust file validation UX

Future Enhancements

  • Team and classroom accounts
  • Crit history archive for users
  • Custom rubric scoring
  • Direct Figma integrations
  • Community or “crit marketplace”
  • Live chat with human reviewers
  • “Designers Online” presence indicator
  • Analytics on designer improvement over time

Example User Journey

  1. Designer visits The Crit landing page.
  1. Clicks CTA to upload work.
  1. Uploads images, PDFs, or web design screenshots.
  1. Enters design goals and optional rubric.
  1. Submits for critique.
  1. n8n-powered multi-agent system analyzes:
    • visuals
    • UX decisions
    • layout and accessibility
    • copywriting
  1. Crit Sheet is generated:
    • praise
    • recommendations
    • educational explanations
  1. Designer reads, saves, or shares feedback.

Voice & Tone

  • Friendly but professional
  • Encouraging, never harsh
  • Educational: explains principles, not just critiques outcomes
  • Conversational yet precise
  • Creative, but not whimsical
  • The Crit is a mentor, not just a critic

Marketing & Brand Strategy

  • Positioning:
    • “AI-powered mentor for design feedback”
    • Not just automated feedback → Educational, professional insights
  • Trust-building tactics:
    • Example Crit Sheet visible before sign-up
    • Testimonials from real designers
    • “Designers Online” presence feature
  • Channels:
    • Design communities (Dribbble, Behance, etc.)
    • Twitter/X (design conversations)
    • LinkedIn (freelancers, educators)
    • YouTube tutorials on critiquing
    • Design schools / bootcamps
  • Long-Term Vision:
    • Become the standard place for rapid, professional design critique
    • Build partnerships with design education programs
    • Evolve into a learning platform, not just a tool

Key Differentiators

  • AI critiques actual visuals, not only text
  • Multi-agent approach = specialized, high-quality insights
  • Educational, mentorship-driven tone
  • Fast, delightful UX for creative professionals
  • Bridges the gap between solo design work and expert feedback

Risks

  • Cost of running vision AI models
  • Hallucinations or low-quality AI feedback
  • Handling large files (e.g. PDFs, big images)
  • Ensuring critiques are genuinely helpful
  • Privacy and security of uploaded designs

Metrics of Success

  • Time-to-crit (speed from upload to output)
  • User satisfaction scores on helpfulness
  • Repeat usage per user
  • Cost per critique
  • Conversion from landing to crit submission
  • Engagement with example Crit Sheets
  • Volume of educational partners or integrations

Vision

“Every designer deserves insightful feedback that makes them better. The Crit turns AI into your design mentor — fast, clear, and ready whenever you need it.”
